In your previous assignment, you implemented a text-based Video Poker game using object-oriented design. Your program ran through the command line and was structured with a clear separation of game logic (e.g., `Card`, `Deck`, `Hand`, `Game` classes).

In this assignment, you will give your Video Poker game a **Graphical User Interface (GUI)** using **Java Swing**. Rather than writing the GUI code from scratch yourself, you will **use ChatGPT to generate the GUI code and integrate it with your existing game logic** through well-crafted prompts.

This assignment emphasizes both **software integration** and **learning how to effectively use AI tools for software development**.

---

## Goals
- Learn how to use Java Swing to build simple GUIs.
- Understand how to integrate a GUI with an existing codebase.
- Practice prompting and refining ChatGPT-generated code.
- Develop critical thinking about AI-assisted software development.

---

## Requirements

1. **Use Java Swing** for your GUI. Other GUI frameworks (like JavaFX) are not allowed for this assignment.
2. You must use **ChatGPT to generate the GUI code**. This includes the layout, display of cards, buttons for holding/discarding cards, updating game state, and so on.
3. You must **integrate the GUI with your existing game logic** (from the previous assignment), and the integration process must also be assisted through ChatGPT prompts. **NOTE:** If you did not successfully complete the homework 8 or are unsure of the correctness of your code, you may substitute in the solutions that we posted on Courseworks for homework 8 and do the GUI development on the solutions instead.

---

## Minimum GUI Features

Your final GUI must:
- Display the current hand of cards using buttons or images.
- Allow players to hold/discard cards with a click.
- Display the player's current credit balance.
- Allow players to start a new hand and play multiple rounds.
- Show messages such as “You won X credits” or “Game Over.”

Submit the following:

## Getting Started

You might begin with a prompt like:

> “Write a Java Swing GUI front end for a Video Poker game. I already have the game logic in classes called `Deck`, `Card`, `Hand`, and `Game`. The GUI should show the player’s hand and allow them to click buttons to hold cards and deal new ones.”

Iterate from there to refine the layout, integrate your own methods, and debug any issues.
